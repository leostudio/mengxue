# 蒙学乐园 UX 优化设计文档

## 概述

对蒙学乐园（mengxue）进行全面的用户体验优化，覆盖五大模块：描红交互、导航流转、庆祝反馈、学习路径、视觉布局。目标用户为幼儿，核心原则是让体验更直观、更有趣、更沉浸。

## 1. 描红交互优化

### 1.1 领路小猫

引入一个小猫角色作为引导者，贯穿整个描红过程。

**行为规则：**
- 每个笔画开始前，小猫出现在笔画起点，面朝笔画方向
- 孩子开始画后，小猫沿正确路径缓慢前进，始终保持在孩子笔迹前方一小段距离
- 小猫的移动速度应略快于预期的书写速度，但不会跑太远——如果孩子暂停，小猫也会停下等待
- 完成一个笔画后，小猫自动跑到下一个笔画的起点（带一个小跑动画），无需任何用户操作
- 全部笔画完成后，小猫进入庆祝动画

**渲染方式：**
- 小猫使用 emoji（🐱）或简单的 Canvas 绘制，保持轻量
- 小猫位于 guide canvas 层，不影响用户在 draw canvas 上的绘制

### 1.2 实时偏离反馈

在描画过程中提供实时反馈，帮助孩子及时修正。

**反馈机制：**
- **小猫回头看：** 当用户笔迹偏离正确路径超过阈值时，小猫停下并转身面向用户笔迹位置，表现出"等等你"的姿态
- **震动提示：** 调用 Vibration API（`navigator.vibrate(50)`），给设备一个轻微短震动
- **提示音效：** 播放一个柔和的提示音（短促、不刺耳）
- **恢复：** 当用户笔迹回到正确路径附近时，小猫转回前方继续前进

**阈值设计：**
- 偏离距离 > 当前笔画宽度的 2 倍时触发反馈
- 反馈有最小间隔（500ms），避免频繁触发造成烦躁

### 1.3 笔画路径数据修复

当前存在笔画引导路径超出字母轮廓的 bug，需要修复。

**修复方案：**
- 逐一审查并校准 26 个字母的 `LETTER_GUIDES` 数据
- 确保所有笔画坐标点落在字母轮廓范围内
- 使用 Canvas 渲染字母后，用 `getImageData` 做像素级验证，确认笔画路径上的点都在字母区域内
- 对于曲线字母（如 S、C、O），增加控制点数量以提高路径精度

### 1.4 无缝笔画衔接

完成一个笔画后，自动过渡到下一个笔画，保持沉浸感。

**过渡流程：**
1. 用户完成当前笔画 → 笔画保留在 draw canvas 上
2. 小猫从当前笔画终点跑到下一个笔画起点（动画时长约 500ms）
3. 下一个笔画的引导线自动出现
4. 进度指示器（圆点）更新状态
5. 全部完成后统一进入庆祝流程

## 2. 导航与页面流转

### 2.1 减少导航层级

去掉字母详情页（detail-screen），简化导航路径。

**改动前：** 首页 → 字母网格 → 字母详情 → 描红/绘本
**改动后：** 首页 → 字母网格 → 描红页（含音频/绘本入口）

**描红页整合：**
- 在描红页面的头部区域，保留返回按钮，并增加音频按钮（🔊）和绘本按钮（📖）
- 音频按钮点击后直接播放当前字母发音
- 绘本按钮点击后跳转到绘本页面
- 删除 detail-screen 相关的 HTML、CSS 和 JS 代码

### 2.2 快捷跳转

完成一个字母的描红后，提供跳转选项。

**完成后界面：**
- 庆祝动画结束后，显示两个按钮：
  - "下一个字母 ➡️"（进入下一个字母的描红）
  - "回到列表 📋"（返回字母网格）
- 按钮使用 bounceIn 动画出现
- "下一个字母"按钮更大更突出，引导继续学习

### 2.3 滑动过渡动画

所有页面切换使用滑动动画，替代当前的瞬间切换。

**实现方案：**
- 进入下一级页面：新页面从右侧滑入（transform: translateX(100%) → translateX(0)）
- 返回上一级页面：当前页面向右滑出（transform: translateX(0) → translateX(100%)）
- 动画时长：300ms，使用 ease-out 缓动
- 使用 CSS transition，不依赖 JS 动画库
- 在 screen 元素上添加 CSS class 控制进入/退出方向

## 3. 庆祝效果升级

保持简单的庆祝系统（不引入积分/勋章），但让效果更丰富有趣。

### 3.1 小猫庆祝动画

- 全部笔画完成后，小猫在画布中央表演庆祝动画
- 动画类型随机选择：跳跃、转圈、打滚等
- 使用 Canvas 逐帧动画或 CSS animation 实现
- 动画时长约 1.5s

### 3.2 多样化鼓励语

替换单一的"太棒了"，改为从鼓励语池中随机选择。

**鼓励语池：**
- "太棒了！"
- "写得真漂亮！"
- "你真厉害！"
- "好棒好棒！"
- "越来越好了！"
- "真是小天才！"
- "完美！"

TTS 语音同步播放选中的鼓励语。

### 3.3 丰富视觉特效

升级庆祝 overlay 的视觉效果。

**特效方案：**
- 使用 Canvas 渲染彩色粒子系统（星星、彩带、小圆点）
- 粒子从中心向四周扩散
- 颜色使用应用现有的彩色调色板
- 特效持续约 2 秒后自然消散
- 保持轻量，不引入第三方粒子库

## 4. 内容与学习路径

### 4.1 自由探索 + 进度可视化

不锁定学习顺序，但在字母网格上展示学习进度。

**进度标记方案：**
- 未学过的字母：保持现有样式
- 已完成描红的字母：卡片右上角显示一个小勾（✓）或小星星（⭐）
- 使用 localStorage 持久化进度数据
- 数据结构：`{ completedLetters: ['A', 'B', ...] }`

**网格布局保持不变，** 孩子可以自由选择任意字母开始学习。

## 5. 视觉与布局调整

### 5.1 视力表页面布局优化

当前视力表页面存在大量空白，需要优化布局。

**改进方案：**
- 减少卡片间距和整体 padding
- 让 4 个方向卡片占据更多可用空间
- 增加引导说明文字或小插图填充空白区域
- 在视力表详情页，调整左右两栏比例，使内容更紧凑

### 5.2 整体布局一致性

跟随前面的改动，统一调整各页面的布局。

**调整范围：**
- 统一各页面的内边距和间距
- 确保新增的导航元素（音频/绘本按钮、完成后跳转按钮）与现有设计风格一致
- 滑动过渡动画在所有页面切换中保持一致的方向和速度

## 技术约束

- 保持原生 HTML/CSS/JS 技术栈，不引入框架
- 所有动画优先使用 CSS transition/animation，Canvas 动画仅用于描红相关
- 使用 localStorage 持久化进度数据
- 使用 Vibration API 实现震动反馈（不支持的设备静默降级）
- 保持移动端兼容性（触摸事件、响应式布局）

## 不在本次范围内

- 积分/勋章/成就系统
- 按难度排序或锁定学习顺序
- 新增字母以外的学习内容
- 底部导航栏
- 后端/数据库
- 用户账号系统
